Q1: Xor

For any given $L$ and $R$, find
$$ \sum_{i=L}^{R} \sum_{j=L}^{R} i \oplus j$$
where $\oplus$ represents the bit-wise xor operation. Mod the answer by $10^9+7$

Input

An integer, $T$, the number of inputs.

The following $T$ lines each include two integers $L$ and $R$.

Output

For each input, an integer, the answer.

Sample IO

Constraint

$1 \le T \le 500$
All $L$ and $R$ are 32 bits integers and for each input $L \le R$

Solution

We can notice for any bit, its contribution is dependent on the number of integers that have $0$ on that bit and the number of those that have $1$ on that bit. Let the former be $x$ and the latter be $y$, the contribution of the $i$-th bit is $2^{i+1}xy$. This is because contribution only occurs when integers have different bits, in other words, contribution can only occur between an element of $X$ and an element of $Y$. This gives us $xy$ pairs that contribute. If we consider the order, thats $2xy$ pairs. Each of them contribute $2^i$ to the answer.

We now need to consider that for a given bit, how many integers have $1$ and how many have $0$. See comments.

Q2: Three-pile Wythoff

Once again, Rose and Scarlet are playing games. This time, the game involves three piles of stones. Each turn, a player can take any amount of stones from any combination of piles. If a player is to take from multiple piles, the number taken needs to be the same for those piles. As usual, Rose is taking the first turn. Determine if she could win regardless of Scarlet's strategy.

Input

In the first line, an integer, $T$, the number of games.

The following $T$ lines each contain three positive integers, $a$, $b$ and $c$, the size of the three piles.

Output

For each game, print "Yes" if Rose can win, else print "No".

Sample IO

Constraints

$1 \le T \le 500$

$1 \le max(a, b, c) \le 300$

Solution

Clearly, $(0, 0, 0)$ is a losing position. We can filter using this the winning positions, since a winning position is one that can get to a losing position in one move. Since the players can only take stones, if we loop the position in dictionary order, then for all position we encounter, its sub-position must have all been encountered. Whenever we encounter a position that has not been marked as winning, it must be a losing position, we can then use it to label other positions as winning.

Q3: Count

Given a binary tree whose nodes are labelled from $1$ to $n$, the node labelled $i$ happens to be the $i$-th node to be visited in the pre-order traversal of the tree. Let $A_i$ be the position of the $i$-th node in the in-order traversal of the tree, we have $m$ limits. The $i$-th limits gives $u_i$ and $v_i$, indicating that $A_{u_i} < A_{v_i}$. Find the number of different binary trees satisfying all above constraints, mod the answer by $10^9+7$.

Input

In the first line, $T$, the number of inputs.

In the first line of each input, $n$ and $m$, meanings per problem description.

In the next $m$ lines of each input, $u_i$ and $v_i$, describing a limit.

Output

For each input output the answer.

Sample IO

Constraints

$T \le 5$, $n \le 400$, $m \le 10^3$

Solution

The constraints related to pre-order traversal are essentially saying that for any sub-tree, the nodes in it are continous and its root has the lowest label. Hence, for any sub-tree of range $i$ to $j$, we can find its answer $f(i, j)$ by,

$$ f(i, j) = 2 \times f(i+1, j) + \sum_{k=i+1}^{j-1} f(i, k) * f(k+1, j) $$

Of course, we need to satisfy the constraints, which can be done using a 2D prefix array.
