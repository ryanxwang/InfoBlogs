Street Construction

Problem Link

Given $k$ parks on $n$ chunks of land, we have $n-k$ chunks of land for building houses, and that we can have at most $k+1$ blocks of houses. The problem hence transform to that of pigeonhole principle, where we have $n-k$ pigeons and $k+1$ pigeonholes.

Cloud Coverage

Problem Link

Let $D_i$ be the distance between the $i$-th person and the $i+1$-th person, then the answer is the minimum consecutive sub-sequence of $D$ containing $k$ elements. Any cloud longer than this would cover more than $k$ people at some point. Clouds shorter than this may work, but we are looking for the maximum length of the cloud.

Janitor

Problem Link

Notice that we only need to pour water on a cell if it has no neighbouring cells higher than it, otherwise it is always better to cover this tiles by covering the tile higher than it. This would allow us to find the answer before replacements are made.

We can of course redo this for every replacement, however, notice that replacements only affect the height of a tile, and that the height of a tile is only affects its neighbours. Hence, for each replacement, we only need to check the tile being changed and its neighbouring tiles.

Detective

Problem Link

We can first make the following observations:

  • Edges of type $1$ imply that two people must both be lying or truthful.
  • Edges of type $2$ imply that two people must cannot be both lying or truthful

This will allow us to two colour each connected component of the graph, and detect some conflicting statements while doing so. Furthermore, this will allow us to group the people, and have the opposition groups - groups that have type 2 edges in between.

We can then proceed to observe that if a group has multiple outgoing type 3 edges, it must be lying and that its opposite group must be truthful. This will allow us to guarentee that some people are innocent and that some people must have stolen the cookie (call them type 1 suspect). If we find multiple people to have stolen the cookie, we detect a confliction. Note that even if there is only one person who must have stolen the cookie, we cannot end our program since there may still be conflicts.

The next observation involves pairs of groups that are opposite to each other, with unknown truthfulness. If both groups in the pair have accusations (we know each of them only have one accusation), and that they accuse different people (otherwise there is a conflict), then one of the people they accuse must have stolen the cookie. This is followed by the observation that if someone is targeted in this fashion twice, then they must be the suspect (call them type 2 suspect). Further thoughts also show us that if multiple pairs of groups are to accuse in this way, there accusations have to all overlap in someway. We can maintain their accusations using std::set.

At last, if both type 1 and 2 suspect exist, they must be the same and not marked as innocent, and we can output the answer, or a conflict occurs. If only one of them exist, they must not be innocent or conflict occurs. If none of them exist, we should look at the suspects maintained by our std::set and output all elements in it that are not innocent. If we have still not outputed anything, we output everyone who is not marked innocent.
